# 架构决策记录（ADR）

本目录记录所有**事后难以逆转或影响深远**的技术与产品决策。长线项目最大的知识流失是「当初为什么这么做」——ADR 就是答案的存档。

## 规则

- 编号递增，文件名 `NNNN-短横线标题.md`，一经合入不可删除；推翻旧决策时新增一篇并在两篇间互相引用。
- 什么情况必须写 ADR：选择/更换框架或核心库、数据模型的结构性变更、权限模型变更、对外 API 破坏性变更、安全与隐私策略变更。
- 状态流转：`提议中` → `已采纳` / `已否决`；被取代时改为 `已取代（见 NNNN）`。

## 模板

```markdown
# NNNN. 标题（动词开头，如「采用 XXX 作为 YYY」）

- 状态：已采纳
- 日期：YYYY-MM-DD
- 关联：（相关 ADR / issue / 文档链接）

## 背景
（是什么问题逼迫我们做决策；约束条件是什么）

## 决策
（我们决定怎么做，写成肯定句）

## 备选方案
（认真考虑过的其它选项，以及为何落选——这是 ADR 最有价值的部分）

## 后果
（正面、负面、需要跟进的事项）
```
