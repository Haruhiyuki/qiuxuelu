# 0001. 记录架构决策

- 状态：已采纳
- 日期：2026-06-10

## 背景

本项目定位为长期演进的公益项目，开发节奏是「持续改进」而非「一次交付」。维护者会更替，AI 辅助开发也需要稳定的决策上下文。如果不记录决策理由，后来者要么盲目推翻、要么盲目遵从。

## 决策

采用轻量级 ADR（Architecture Decision Records）流程，存放于 `docs/adr/`，规则见该目录 README。每个结构性决策（框架选型、数据模型、权限模型、对外契约）合入前必须附带 ADR。

## 备选方案

- **不记录，靠 git log 与注释**：git log 记录「做了什么」，无法记录「为什么不做别的」，落选。
- **重型 RFC 流程**：对 1-3 人团队过重，落选；若未来社区化开发，可在 ADR 之上叠加 RFC。

## 后果

- 正面：决策可追溯，新人/AI 上手成本低，避免重复争论。
- 负面：合入前多一步写作成本——刻意为之的摩擦。
