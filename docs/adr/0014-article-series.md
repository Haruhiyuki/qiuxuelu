# ADR-0014：文章系列（series）

状态：已接受
日期：2026-06-15
关联：ADR-0007（页面模式）、ADR-0008（作者完整协作权）、ADR-0003（修订模型）

## 背景

作者常把多篇文章组织成一个有序整体（教程合集、连载、专题）。此前只有「板块 + 标签」两套
横向分类，缺少作者主导的、有序的纵向编排。需要引入「文章系列」让作者新建系列、把已有文章放进去、
在系列内新建文章、调整顺序；读者在文章底部看到所属系列并可进入系列页顺序阅读。

## 决策

### 数据模型（新增两表，不触碰修订模型红线）

- `series`：`id` / `owner_id`(→user) / `slug`(unique，nanoid) / `title` / `description?` / 时间戳。
  系列是**作者的编排元数据**，不含正文、不进内容寻址链，与 blobs/revisions/document_refs 完全解耦。
- `series_items`：`document_id`(**主键**，→documents cascade) / `series_id`(→series cascade) /
  `position` / `added_at`。
  - **document_id 作主键 = 一篇文章至多属于一个系列**（与「文章底部显示其所属*某个*系列」一致；
    多系列归属留待将来按需扩展，届时改 PK 并新增 ADR）。
  - `position` 为系列内顺序（不强制唯一，重排时整体改写），`(series_id, position)` 建索引。

### 权限（自归属，不新增治理能力）

系列是与「收藏 / 个人资料 / 教育经历」同类的**作者自归属编排**，不是内容治理对象，故**不**新增
`Capability`、不走 `can()`（与 reactions/account 既有自归属动作一致）；改为在 Server Action 内做
**所有权直检**：

- 建/改名/改简介/删/重排系列：要求 `series.owner_id === actor.id` 且账号未停用。
- 把文章加入系列：额外要求 `document.owner_id === actor.id`——**只能编排自己的文章**
  （公共页文章的 `owner_id` 仍是原作者，原作者保留编排权，符合 ADR-0007 保留作者身份）。

> 例外仍受治理约束：被停用账号（`status='suspended'`）一律拒绝（与既有自归属动作同闸）。

### 可见性

- 系列页 `/series/[slug]` 公开可看：列出系列内**已发布**文章（按 position 排序）；
  **系列所有者本人**额外可见草稿/审校中条目并带状态标注。
- 个人主页 `/u/[id]`：展示该用户「含已发布文章」的系列；所有者本人可见全部。
- 文章页底部：若文章属于某系列，显示系列导航卡（系列名 + 第 N/共 M 篇 + 上一篇/下一篇，
  邻篇只在已发布范围内取）。

### 派生与一致性

`series_items` 是可变编排指针，无历史不变式；文章删除走外键级联，下线/转草稿不删成员、仅在
非所有者视图中按发布状态过滤。系列不进搜索索引与知识图谱（编排非内容）。

## 取舍与后果

- 一篇文章单系列：模型与 UI（单选、底部单卡）都更干净；多归属是后续可叠加的扩展点。
- 自归属直检而非 `can()`：避免为非治理编排撑大鉴权矩阵；与既有 reactions/account 动作风格一致，
  红线「新 Server Action 必须先鉴权再干活」以所有权直检满足。
- 系列与板块正交：一篇文章既属某板块（必填）又可选属某系列，互不影响。
