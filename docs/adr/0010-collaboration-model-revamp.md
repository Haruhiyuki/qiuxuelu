# ADR-0010：协作模型重构（修订 / 修订申请 / 编辑建议 三件套）

状态：已接受（分三期实施）
日期：2026-06-13
关联：ADR-0004（建议=分支）、ADR-0005（双线权限）、ADR-0007（页面模式）、ADR-0008（作者完整协作权）

## 背景

原协作语义混乱：「编辑建议」(suggestion 分支) 与「协作编辑/编辑申请」(doc.edit_direct) 名实不符，且没有「不改内容、只提意见」的轻量反馈通道。本 ADR 把协作重新明确为三件套，并据页面公开性重排权限矩阵。

## 决策

### 1. 三种协作机制（注意名称整体换位）

| 名称 | 含义 | 直接改内容 | 生效 | 对应实现 |
|---|---|---|---|---|
| **修订** | 直接改文章，**立即生效**，权限者可撤回 | 是 | 立即（进巡查） | 现 `doc.edit_direct` |
| **修订申请** | 直接改文章，但**需权限者审核**才生效 | 是 | 审核后合入 | 现 suggestion 分支（`suggestion.*`） |
| **编辑建议** | **不改内容**，对全文/片段提意见 → 送作者+编辑后台，权限者处理后置「处理状态」+回复 | 否 | —（意见） | **新建 `feedback.*` + 表** |

### 2. 权限矩阵（非作者，按页面公开性）

| 能力 | 🌐 公共页 | 🔒 私有页 |
|---|---|---|
| 编辑建议 | T1 | T2 |
| 修订申请 | T2 | T3 |
| 修订 | T3 | 权限者 |

- **权限者** = 作者本人 + 板块版主及以上角色（含编辑在内的治理角色 / owner）。
- 修订（`doc.edit_direct`）：公共页 T3+（即时生效+巡查），私有页仅权限者——与 ADR-0007 现状一致，无需改。
- 修订申请（`suggestion.create`）：**改为按可见性分级**——公共页 T2、私有页 T3。
- 编辑建议（`feedback.create`，新）：公共页 T1、私有页 T2。

### 3. 公开性与公示页

- **私有页**：编辑建议 / 修订申请 **不公开**；**修订记录公开**（但无评论/打分）。
- **公共页**：编辑建议 / 修订申请 / 修订记录 **全公开**，并提供**专门公示页**可**评论 + 打分**；评论打分**同步到权限者后台**供处理参考。

### 4. 统一入口「协作」按钮

文章顶部「协作」按钮 → 小弹窗列出三功能；权限不够的**标灰 + 显示原因**；点可用项进入对应使用界面。（替换 ADR-0008 时期加的「协作编辑/提建议」chip。）

### 5. T4 重新明确

T4 额外权限 = **处理公共页修订申请** + **管理修订**（取代原 `suggestion.review` 的笼统语义）。

### 6. 其它权限/升级调整

- **T0 起**：文中发图（`media.upload` 降到 T0）+ 完整文章编辑能力。
- **T2+**：发文**免预审**（自助发布，不经编辑审批）。
- **升级**：注册=T0，**发首文=T1**，之后靠贡献**积分**（积分细则另设计）。

## 实施分期

- **一期**（本次）：三件套重命名 + 权限矩阵（修订申请按可见性分级、media 降 T0）+ 协作弹窗入口 + T0 发图/完整编辑。`feedback`（编辑建议）仅在弹窗占位（标「二期」）。
- **二期**：新建「编辑建议」反馈系统（全文/片段意见 + 处理状态/回复后台 + 个人中心打通）+ 公共页公示页（评论/打分，同步后台）。
- **三期**：信任升级改积分制 + T2 免预审 + 发首文=T1。

## 取舍与后果

- 内部标识保持稳定：一期只换**用户可见文案**与权限门控，`suggestion.*` 能力名 / `/suggest` 路由 / DB 枚举不动，降低风险；新「编辑建议」用独立 `feedback.*` 命名空间，无碰撞。
- 取代 ADR-0007 §2 矩阵中修订申请（suggestion）的「TL2 一刀切」为按可见性分级；其余沿用。
- 二期「编辑建议」不进修订模型（纯意见，不产生 revision），不触碰内容寻址红线。
