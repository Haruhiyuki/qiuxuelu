# 权限等级参考（活文档）

> 以代码为准：能力清单见 `packages/domain/src/authz/capabilities.ts`，判定逻辑见 `can.ts`。
> 关联：ADR-0005（双线权限）、ADR-0007（页面模式）、ADR-0008（作者完整协作权）、ADR-0009（AI 评论审核）、ADR-0010（协作三件套重构）。
> 本文是**活文档**（随代码更新）；ADR 是某次决策的时点记录。

## 协作三件套（ADR-0010）

| 名称 | 含义 | 改内容 | 生效 | 内部实现 |
|---|---|---|---|---|
| **修订** | 直接改文章，**立即生效**，权限者可撤回 | 是 | 立即（进巡查） | `doc.edit_direct` |
| **修订申请** | 直接改文章，但**需权限者审核**才生效 | 是 | 审核后合入 | suggestion 分支（`suggestion.*`） |
| **编辑建议** | **不改内容**，对全文/片段提意见 → 送作者+编辑后台处理 | 否 | —（意见） | `feedback.*` |

入口：文章顶部「协作」按钮 → 弹窗列三项，权限不够的标灰+原因。

## 骨架：两条线 + 一条红线

- **信任线（晋升给能力）**：TL0→TL4，随贡献**自动**解锁，给「能做事」的能力。
- **角色线（任命给权力）**：editor→section_mod→admin→superadmin，**人工授予**，给「管事」的权力。`editor` / `section_mod` **带板块作用域**（仅在被授板块生效），`admin` / `superadmin` **全局**。
- **红线 `ROLE_ONLY_CAPS`**：一组「权力型」能力**永不经信任线**，只能靠任命角色（见下）。

判定顺序（`can()` 唯一入口）：**停用 → 制裁 → 角色线 → 作者特例 → 信任线**。返回的是「**裁决 + 义务**」，不是布尔（如公共页修订附带 `enqueue_patrol` 进巡查）。

## 信任线 TL0–TL4（累计；括号为该级新解锁）

| 等级 | 名称 | 累计能做 |
|---|---|---|
| **TL0** | 新成员 | 阅读、**建草稿/发文/文中发图（完整编辑能力，`media.upload` 降到 T0）**、举报；**发表评论**（AI 秒审，无预审/无限速，ADR-0009） |
| **TL1** | 成员 | ＋**行内批注**(`comment.inline.create`)、对**公共页**文章提**编辑建议** |
| **TL2** | 贡献者 | ＋对**公共页**文章提**修订申请**(`suggestion.create`)、对**私有页**文章提**编辑建议** |
| **TL3** | 资深贡献者 | ＋在**公共页**直接**修订**他人文章(`doc.edit_direct`，即时生效＋进巡查)、对**私有页**文章提**修订申请** |
| **TL4** | 共建者 | ＋**处理公共页修订申请、管理修订**(`suggestion.review`)；无自动路径，由社区提名 + 管理员人工授予 |

> 升级（ADR-0010）：注册=T0，**发首文=T1**，之后靠贡献积分（积分细则三期设计）。

## 角色线（累计；每级 = 前一级 ＋ 下列）

| 角色 | 作用域 | 新增权力 |
|---|---|---|
| **editor 责任编辑** | 板块 | 含信任线到 TL4 的全部 ＋ **发布/审校通过**(`doc.publish`)、**回退**(`doc.rollback`)、**认领巡查队列**(`queue.claim`)、**审校＋合并建议**(`suggestion.review`/`merge`) |
| **section_mod 板块版主** | 板块 | ＋评论管理(`comment.moderate`)、**处理举报**(`flag.review`)、**下架**(`doc.unpublish`)、**锁定编辑**(`doc.protect`，二元 open/locked，见 ADR-0011)、**精选**(`doc.feature`)、**设为公共页**(`doc.set_visibility`) |
| **admin 管理员** | 全局 | ＋**封禁**(`user.suspend`)、**调信任等级**(`user.trust_adjust`，含授 TL4)、**授板块角色**(`role.grant_section`)、**管板块**(`section.manage`)、**发近闻/公告**(`announcement.manage`) |
| **superadmin 超管** | 全局 | ＋**授全局角色**(`role.grant_global`)、**系统配置**(`system.config`) |

> editor 例外：在**私有页**且非自己的文章上，editor **不获**「审核/合并建议」权（私有页的建议管理权只归作者；section_mod 及以上保留治理监督）。

## 红线 `ROLE_ONLY_CAPS`（信任线永远拿不到，只能任命）

`doc.publish`、`doc.unpublish`、`doc.protect`、`doc.feature`、`doc.set_visibility`、`flag.review`、`user.suspend`、`user.trust_adjust`、`role.grant_section`、`role.grant_global`、`section.manage`、`announcement.manage`、`system.config`。

一句话：**发布、下架、锁定、精选、设可见性、处理举报、封人、调级、授角色、管板块、发公告、改系统**——全是任命来的权力，攒信任攒不到。

## 作者特例（对自己的文档，ADR-0008）

作者从 **TL0** 起就对自己的文章拥有完整协作权：

- **直接编辑**(`doc.edit_direct`)、**提交审批**(`doc.submit`)；
- **审核 + 合并**他人提交的修订申请(`suggestion.review` / `suggestion.merge`)。

仍受约束：制裁仍一票否决；`edit_policy='locked'`（管理员锁定编辑，ADR-0011）下作者也不能直编；首次**发布**仍需编辑审校（`doc.publish` 是红线，不自助）。

## 协作他人文章：公共页 / 私有页 两条线（ADR-0010）

三种协作方式对**他人**文章的门槛按页面公开性分级（能力阶梯：**编辑建议 < 修订申请 < 修订**）：

| 协作方式 | 说明 | 🌐 公共页 | 🔒 私有页 |
|---|---|---|---|
| **编辑建议** | 提意见、不改内容 | T1 | T2 |
| **修订申请** | 改内容，需审核才生效 | T2 | T3 |
| **修订** | 改内容，立即生效（可撤回） | T3 | **权限者** |

- **权限者** = 作者本人 + 板块版主及以上。私有页的「修订」只归权限者；编辑（editor 角色）在私有页可直接修订（角色线）。
- **作者本人**：对自己的文章 TL0 起完整协作权（直接修订 + 审核/合并他人的修订申请，ADR-0008）。
- **公开性**（ADR-0010 二期落地）：私有页的编辑建议/修订申请不公开、修订记录公开（无评论打分）；公共页三者全公开 + 公示页可评论打分、同步后台。
- **私有 → 公共升级**：私有页累计「他人贡献」≥ 阈值（`site_settings['doc.publicize'].threshold`，默认 20）自动升级，或管理员 `doc.set_visibility` 手动升级；保留原作者身份。

## 制裁（一票否决，压过角色——治理红线高于职务）

| 制裁 | 封锁 |
|---|---|
| `silence` | 评论（含行内） |
| `no_suggest` | 提修订申请 |
| `no_edit` | 建草稿 / 提交 / 直编 / 合并建议 / 传图 |
| `suspend` | 除阅读外全部 |

账号 `status=suspended` 早于一切判定（停用的 admin 也无权）。制裁可带板块作用域（仅该板块生效）或全站。
